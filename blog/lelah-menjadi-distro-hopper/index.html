Lelah Menjadi Distro Hopper

Wed, 31 Jul 2024

Sedikit cerita, aku memulai projek untuk website ini pada bulan April 2024. Eleventy menjadi pilihan karena SSG (Static Site Generator) Gatsbyjs yang ku gunakan untuk website-ku yang sebelumnya, "Aflasio" bisa ku bilang agak payah.

Bukan maksud menjelekkan Gatsbyjs sih ya, tapi gini maksudnya. Terakhir kali aku nge-blog, mungkin sekitar 2019 kalo diliat dari postingan terakhirku di sana, semuanya aman aja. Nodejs aman, build aman. Sampe tiba-tiba aku stop pake linux, berhenti nggambar dan fokus ke kerjaan utama.

Nah masalahnya dimulai awal tahun ini, 2024. Aku tiba-tiba ada niat mau nge-blog lagi. Ku download tuh ubuntu, trus install dualboot. Langsung pasang vscode, nodejs, git. Lalu clone itu project. Eh... Deprecated banyak, sampe gak bisa build. Buset dah... Perasaan baru aja ditinggal berapa tahun.

Dari sini mulai nyari-nyari SSG yang bisa tahan lama dan juga fleksibel soal permalink, dan aku butuh yang gak banyak pake JS. Ketemulah Eleventy yang ku pake sampe sekarang.

Wait, ini koq jadi ngomongin SSG sih? Kan judulnya tentang distro hopper?

KDE neon

Oh iya, kembali ke masalah. Sebenarnya sebelum menginstall Ubuntu pada April 2024 ini. Aku udah coba KDE Neon karna emang itu distro terakhir yang ku pake 2019. Tapi nih, ada tapinya... Koq UI default-nya berasa jadul ya? Yha itu perspektif orang sih. Ada yang gak mentingin UI dan lebih mentingin fungsi. Tapi ya, kalo aku pribadi. UI itu diliat tiap kali kita buka sistem. Kalo jelek, yha gimana mau betah lama-lama sama itu sistem.

Ubuntu

Akhirnya terpilihlah Ubuntu 24.04 yang kebetulan baru release. So far so good. Bahkan di laptopku yang pake Intel Core I5 10Gen dengan ram 8GB. Nah selang beberapa minggu setelah update, ketemu masalah baru. Layar tiba-tiba flicker sendiri. Nyari solve di forum ubuntu sampe ke grup telegram gak ngefek. Dan mulailah menjadi "Distro Hopper".

Fedora

Ku beranikan diri pake Fedora Workstation. Impresi pertama rasanya koq ini lebih ringan dibanding Ubuntu, padahal sama-sama pake gnome. Cuman ketemu nih masalah baru. Gak tau apa jaringanku yang bermasalah apa sistem DNS yang bermasalah. Ini fedora gak bisa buka situs google.com padahal itu ku pake untuk search engine utama. Gapapa sih, masih bisa diakali dengan cara ganti DNS. Selain itu aku juga ketemu masalah touchpad yang scroll-nya terlalu cepat. Tapi bisa juga diakali sampe ku bikin catatan di postingan ini. Sempat lama lho pake ini distro. Distro joss lah pokoknya. Kembali ke masalah, ya masalah. Karna tiap akhir paragraf bakal ku kasi tau masalahnya wkwkwk.

Masalahnya yha bosan aja.

Giliran sistemnya aman, eh malah user-nya yang bosan.

Manjaro

Pindah OS lagi, clean instal. Pake Manjaro broo... Eh malah kena bully di grup, katanya distro ampas. Ampasnya dimana? Perasaan baik-baik saja. Rupanya, eh rupanya. Emang bener baru selang berapa jam install ketemu bug password salah, gak bisa login. wkwkwk

Arch Linux

Balik lagi nih nyoba Ubuntu, kan kirain masalah flickering screen udah dapat update dan solved gitu. Eh rupanya belum. Panass... Akhirnya nekad download iso Arch Linux. Padahal baru 1 jam clean install Ubuntu.

Pengalaman install distro yang satu ini, aku sempat gagal sekitar 4 kali. 2 kalinya install manual ngikutin docs di situs Arch, 2 kalinya lagi pake archinstall. Gagalnya juga sama, gak bisa booting. Nah setelah install ke 5 kalinya aku baru sadar pas tekan tombol F12, koq ada Windows dan Linux terpisah? Lalu ku arahkan hint ke linux. Tadaaaa booting ke Arch. Jadi 4 kali install ulang itu sebenarnya berhasil, gak tau aja kalo harus booting manual. Kocak gak tuh?

Sempat lama nih pake Arch, bangga banget rasanya. Akhirnya bisa pake meme:

I use Arch btw

Sampe bikin custom sticker di Telegram. Dan bukannya aman-aman aja sih, banyak juga permasalahan tapi masih bisa solved sampe jadi beberapa catatan di sini. Masalah utamanya balik lagi nih. Giliran sistem aman, orangnya yang gak aman. Bosen, trus oprek tuh. Install ini itu akhirnya ada crash. Lupa paket apa aja yang udah di install, males benerin. Install ulang. Setelah install ulang, janji gak bakal oprek lagi. Halah, janji sama diri sendiri aja gak bisa nepatin. Install ulang lagi lah...

Sampe akhirnya 3 hari yang lalu tanggal 29 Juli 2024, baru kelar upgrade. Jrenggg gak bisa booting grub_resque. Duh gini amat pake Linux. Kalo orang bilang,

Mendingan pake Windows

Yha aku emang pake Windows sebagai OS utama karna laptop yang ku pake ini laptop kantor.

Sebenernya aku pake Linux cuman untuk ngoding aja. Palingan VSCode, nodejs, dan git, udah. Masalahnya ngoding di Windows itu rada aneh, apalagi nyentuh terminal. Yha terminal windows juga bagus koq. Gimana ya? Tetep aja rasanya ada yang aneh. Makanya kekeh pake Linux biarpun dualboot. Tapi hanya gegara hal yang ku ceritain di atas malah menuntun menjadi seorang distro hopper.

Atomic & Nix

Kepikiran untuk nyobain Atomic, mungkin aku butuhnya yang stabil gak banyak bug, error ini itu. Ku cobain tuh Fedora Silverblue. Baru sejam abis install langsung sadar. Gimana caranya aku setel kecepatan scroll touchpad woi, ini sistem gak bisa dioprek. Aaaaaaa....

Beli paket data lagi, kebetulan aku pake Mifi (Mobile Wifi). Trus download Nix OS. First impression, keren banget ini OS. Berasa mainan git yang branch-nya bakal muncul otomatis di bootloader.

Ada lagi yang bikin hal...

Sistemnya gak bisa dioprek, bangke! Masalahnya aku butuh setel touchpad lagi. Ikh... geram loh, di Nix OS itu gak bisa pake cara yang biasa ku pake di distro lain. Tiap reboot, setelan touchpad balik lagi ke asal.

Akh kampreeeetttt.
Dahlah, balik ke Ubuntu aja...

Baru kelar install Ubuntu tadi pagi jam 5an. Ini lagi solving gimana ngatasin flickering screen. Capek juga ngetik panjang lebar. Kalo ini sistem udah bener, gak bakal install distro lain lagi. Sekian...

Terimakasih udah mampir.
